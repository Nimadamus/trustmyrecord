How It Works

Inside the TrustMyRecord MLB Simulator

What the simulator is built to do

The MLB Simulator plays each game out one plate appearance at a time. Every batter faces the selected pitcher with outcome probabilities (walk, strikeout, single, double, triple, home run, out) built from real rate stats and combined through the odds-ratio method, then runners advance through a base-and-out state machine. The box score, win probability, run distribution, strikeouts, walks, and home runs all emerge from those simulated events rather than being assigned after the fact.

Team run output is anchored to a structured expected-run model that already accounts for park, starter and bullpen quality, era context, and verified live inputs when available, so totals stay realistic while each simulated game varies the way real baseball does. It is not a guarantee; the output is a model-based estimate that should be read as scenario guidance, not certainty.

Current, classic, and mixed era modes

Current Teams mode compares modern MLB baseline profiles. Classic Teams mode uses curated historical team profiles and normalizes them into a shared simulator environment. Mixed Era Matchup lets a current club face a classic team, with era adjustment shown clearly in the projection.

When verified live context is unavailable, the simulator stays usable with baseline ratings instead of inventing unavailable injuries, weather, confirmed starters, sportsbook odds, or betting edges.

Why starter selection matters

Starting pitching is one of the biggest drivers of a baseball game projection. The starter dropdown changes both run expectation and win probability, so users can compare how different rotation options affect the matchup.

Current-team player names are loaded from the MLB Stats API active roster endpoint for the selected team, fetched with no-store behavior and a simulator build cache-buster. If that roster cannot be verified with enough hitters and pitchers for the selected team, the player box score says the roster is temporarily unavailable instead of filling rows with fallback names.

How to read the output

Use the projected score range, expected runs, win probability, and key explanation together. A higher win probability means the model sees a stronger matchup profile, not that the result is guaranteed.

For fans, it is a way to explore matchups. For handicappers, it can be one decision-making input alongside market prices, lineup news, weather, injury reports, and personal judgment.
