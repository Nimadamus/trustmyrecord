How It Works

Inside the TrustMyRecord MLB Simulator

What the simulator is built to do

The MLB Simulator models baseball matchups using structured team baselines, starting pitcher selection, era context, and available verified live inputs when they can be matched. It is designed to produce a directional projection: expected runs, score range, win probability, matchup notes, and the factors driving the result.

It is not a random number generator, and it is not a guarantee. The output is a model-based estimate that should be read as scenario guidance, not certainty.

Current, classic, and mixed era modes

Current Teams mode compares modern MLB baseline profiles. Classic Teams mode uses curated historical team profiles and normalizes them into a shared simulator environment. Mixed Era Matchup lets a current club face a classic team, with era adjustment shown clearly in the projection.

When verified live context is unavailable, the simulator stays usable with baseline ratings instead of inventing unavailable injuries, weather, confirmed starters, sportsbook odds, or betting edges.

Why starter selection matters

Starting pitching is one of the biggest drivers of a baseball game projection. The starter dropdown changes both run expectation and win probability, so users can compare how different rotation options affect the matchup.

Starter lists are simulation inputs unless a probable starter is explicitly matched from a verified live source. If a field like WHIP, handedness, or win-loss record is not available, it is not fabricated.

How to read the output

Use the projected score range, expected runs, win probability, and key explanation together. A higher win probability means the model sees a stronger matchup profile, not that the result is guaranteed.

For fans, it is a way to explore matchups. For handicappers, it can be one decision-making input alongside market prices, lineup news, weather, injury reports, and personal judgment.
