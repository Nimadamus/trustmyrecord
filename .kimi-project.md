# TrustMyRecord - Project Context for Kimi Agent

## Project Overview
TrustMyRecord is a social sports betting tracking and analytics platform.
Users log picks, track performance, follow other bettors, compete in challenges,
and get advanced statistics on betting patterns.

## Database Schema (PostgreSQL)
Key tables:
- **users**: id, username, email, password_hash, avatar, bio, poll_points, trivia_points, etc.
- **picks**: id, user_id, sport, team, opponent, pick_type, odds, units, stake, result, status, confidence, event_date
- **competitions**: contests users join
- **challenges**: 1v1 challenges between users
- **polls / poll_votes**: Community polls
- **trivia_games / trivia_questions / trivia_results**: Trivia system
- **conversations / messages**: Private messaging
- **follows**: follower_id, following_id
- **notifications**: user_id, notification_type, title, message, is_read

## API Endpoints (Backend Routes)
Auth: POST /api/auth/register, /login, /verify-email, /forgot-password, /reset-password, /refresh, /logout
Profile: GET /api/profile/:username/stats, /friends, /competitions, /challenges, /poll-trivia-stats
Challenges: GET/POST /api/challenges, POST /api/challenges/:id/accept, /decline, /complete
Polls: GET /api/polls, POST /api/polls/:id/vote, GET /api/polls/:id/results
Trivia: GET /api/trivia/games, POST /api/trivia/games/:id/submit
Messages: GET /api/messages/conversations, POST /api/messages/conversations/direct
Picks: POST/GET /api/picks
Social: POST/DELETE /api/users/:id/follow, GET /api/users/:id/followers
